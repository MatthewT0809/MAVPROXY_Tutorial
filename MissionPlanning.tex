\section{Drone Setup}
In this section, things tends to be more bespoke and my instructions will start to become more arbitary and generalised as the configuration differs for each drone. To appericate the process however, this section provides a working example of a functional commmunication link with the highlighted components used.  

\label{sec:hostport}
In this guide, an ESP8266 Wifi Module \footnote{ESP8266 is reported to have connection issues in swarms, use raspberry pi if doing swarm flights.}, a Pixhawk6C flight controller\footnote{You are more than welcome (encouraged even) to explore using host wifi features from raspberry pi, which I have been told to be more stable than the ESP8266.}\footnote{Do note that not all Pixhawk works the same when setting this up, I have had no success with older versions of Radiolink Pixhawk with this configuration, also I have not tried this with PX4.}, a Radio Controller/Reciever configuration, IR Markers, and a custom drone is used. \subsection{Setting up the ESP8266}
To set up the ESP8266, power up the Pixhawk (Remove the propellers). \footnote{If the ESP8266 is being used for the 1st time, please see \cite{engineeringprojects_esp8266} for setting up wifi name and password.} \footnote{If you are using the drone after the 2025/2026 cohort, it is named as Inspector/Surveyour with password being inspector/surveyor correspondingly} and connect your GCS to the wifi being hosted by the ESP8266. The wifi connection should be just a conventional wifi network as shown in figure \ref{fig:wifi1}. Connect to it and enter the password like a normal wifi network.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\linewidth]{Images/Wifi1.png}
    \caption{Wifi Network Hosted by ESP8266.}
    \label{fig:wifi1}
\end{figure}

Once you are connected to the WiFi, you need to find out the IP address of the WiFi network, this is known by going into the wifi settings in \ref{fig:wifi3}: 

\begin{figure}[H]

    \includegraphics[width=0.8\linewidth]{Images/Wifi3.png}

\subsection{Mavproxy Telemetry}
To test if the telemetry communication link is working, return to the GCS and type in the mavproxy command (Remeber to select the ESP8266 Wifi Network). If successful, telemetry should start showing up like figure \ref{fig:GCST3}.

\begin{lstlisting}
    mavproxy.py --master :14554[Replace with your port number here] --console --map
\end{lstlisting}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\linewidth]{Images/GCSTelem3.png}
    \caption{Telemetry Data Showing up in Mavproxy Console.}
    \label{fig:GCST3}
\end{figure}
Recalling the Host Port number from before in figure \ref{fig:wifi6}, make sure to replace 14554 with the number of your port. 
\subsection{IR Markers}
Now your GCS connetions are all done, we are moving towards the final  steps before actually flying the drone with VICON! Mount the IR markers with hot glue on location of your drones that is easily seen by the cameras. Have at least 4+ per drone and place them in an assymptical arragement, an example is shown in figure \ref{fig:irmarkers} below:  

\begin{figure}[H]
    \centering
    \includegraphics[width=1\linewidth]{Images/DSC05445.JPG}
    \caption{Example of Drone IR Markers layout.}
    \label{fig:irmarkers}
\end{figure}

If you are flying more than 1 drone, your assortment of IR markers must be geometrically different, otherwise the Vicon System may mistake the wrong object to track. (This has happened to me before and caused a crash.) 

\subsection{Final Intergration}
Finally, let's test the overall system. Plug in your GCS to the Vicon ethernet port as seen in figure \ref{fig:hardware}, connect the GCS laptop to the ESP8266's wifi, and place your drone inside the flight arena. \\
\vspace{0.2cm}
Setup the Vicon system by following section \ref{sec:viconsetup} above. This includes turning on the cameras, creating a mask, calibrating cameras, setting the origin, and define your now ready to go drone as a new object in section \ref{sec:defineNewObject}. \\
\vspace{0.2cm} 
One last thing, we must let pyvicon know the name of the object so it knows which object to track. To do this, go into the to GCS's venv, and navigate to \textit{~/miniconda3/envs/VICON/lib/python3.1/site-packages/MAVProxy} and open the file named \textit{mavproxy\_vicon.py}, as shown in figure \ref{fig:NC1}:

\begin{figure}[H]
    \centering
    \includegraphics[width=1\linewidth]{Images/NameChange1.png}
    \caption{Location of mavproxy\_vicon.py.}
    \label{fig:NC1}
\end{figure}

Align your focus on line 29 to 36, and change \textit{(origin\_lat)(origin\_lon)(origin\_alt)} to 0 (Otherwise your drone will fly towards Australia, where the origin is defaulted at the glorious Canberra Model Aircraft Club Flying Field), \textit{(object\_name)} as your object name. (Here, \textit{MATT\_INSPECTOR} is the name of my drone as defined within the Vicon Host PC). Press \textit{Save} on top right to save, as shown below in figure \ref{fig:NC2}. 

\begin{figure}[H]
    \centering
    \includegraphics[width=1\linewidth]{Images/NameChange2.png}

Once completed, activate MAVproxy console, once again, this is done by opening terminal and typing:
 \begin{lstlisting}
    mavproxy.py --master :14554 --console --map
\end{lstlisting}

Within mavproxy, type the following to load the Vicon Module: 

 \begin{lstlisting}
    module load vicon
\end{lstlisting}

 \begin{lstlisting}
    vicon set
\end{lstlisting}

 \begin{lstlisting}
    vicon start
\end{lstlisting}

You should have the same results as figure \ref{fig:Works1}. A good way to verify if the object is being tracked correctly within the scirp is when entering \textit{vicon start}, it will only return the statement \textit{Connected to subject "MATT\_INSPECTOR" segment "MATT\_INSPECTOR} when it is working correctly.

When console mentions height is at -12 while your drone is on the ground, this is referring a 12cm offset from the bottom to the center of the drone. The position of the drone is defined as the geometric center of the assortment of IR markers you placed.
